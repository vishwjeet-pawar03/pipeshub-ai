## 취약점 신고

우리는 보안 취약점을 매우 중요하게 여기며, 발견한 문제를 책임감 있게 공개하려는 여러분의 노력에 감사드립니다.

### 신고 방법

**보안 취약점에 대해 공개 GitHub 이슈를 생성하지 마세요.**

대신 다음 주소로 이메일을 보내 보안 취약점을 신고해 주세요: **abhishek@pipeshub.com**

### 포함할 내용

취약점을 신고할 때는 다음을 포함해 주세요:

- 취약점 설명
- 문제 재현 단계
- 잠재적 영향 평가
- 제안하는 수정 방안(있는 경우)
- 후속 연락을 위한 귀하의 연락처

### 예상되는 절차

- **초기 응답**: 48시간 이내에 신고 접수를 확인합니다
- **평가**: 영업일 기준 5일 이내에 초기 평가를 제공합니다
- **업데이트**: 해결될 때까지 7일마다 진행 상황을 알려드립니다
- **해결**: 중대한 취약점은 30일 이내에 해결하는 것을 목표로 합니다
- **공로 인정**: 보안 권고문에서 귀하의 기여를 밝힙니다(익명을 원하시는 경우 제외)

### 공개 일정

- **0일차**: 취약점 신고
- **1~2일차**: 초기 확인
- **1~5일차**: 초기 분류 및 영향 평가
- **1~30일차**: 수정 개발 및 테스트
- **30일차 이후**: 수정 배포 후 공개하며, 일반적으로 최초 신고로부터 90일 이내입니다.

### 취약점 평가

우리는 다음 기준에 따라 취약점을 분류합니다:

- **심각(Critical)**: 사용자 데이터 또는 시스템 무결성에 대한 즉각적인 위협
- **높음(High)**: 광범위한 사용자 노출을 동반한 중대한 보안 영향
- **중간(Medium)**: 제한적 노출을 동반한 보통 수준의 보안 영향
- **낮음(Low)**: 영향이 미미한 경미한 보안 문제

### 보안 모범 사례

이 프로젝트에 기여할 때:

- 의존성을 최신 상태로 유지하세요
- 안전한 코딩 관행을 따르세요
- 모든 사용자 입력을 검증하세요
- 적절한 인증 및 권한 부여를 사용하세요
- 보안 이벤트에 대한 로깅을 구현하세요
- 정기적인 보안 테스트를 권장합니다

### 범위

이 보안 정책은 다음에 적용됩니다:

- 메인 애플리케이션 코드
- 공식 Docker 이미지
- 보안에 영향을 줄 수 있는 문서
- 우리가 직접 유지 관리하는 의존성

### 범위 외

다음은 일반적으로 보안 취약점으로 간주되지 않습니다:

- 서드파티 의존성의 문제(해당 유지 관리자에게 신고해 주세요)
- 사회 공학적 공격
- 물리적 보안 문제
- 증폭 없이 자원 고갈을 통한 서비스 거부

### 세이프 하버

우리는 다음 조건을 충족하는 보안 연구자에게 세이프 하버를 제공합니다:

- 개인정보 침해 및 서비스 중단을 피하기 위해 성실히 노력하는 경우
- 본인이 소유한 계정 또는 명시적 허가를 받은 계정과만 상호작용하는 경우
- 다른 사용자의 데이터에 접근하거나 수정하지 않는 경우
- 취약점을 신속히 신고하는 경우
- 해결되기 전에 취약점을 공개하지 않는 경우
