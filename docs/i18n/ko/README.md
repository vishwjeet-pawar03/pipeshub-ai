<h2 id="about-pipeshub">PipesHub — 설명 가능하고 확장 가능한</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> 는 기업 지식을 연결하고, 인용이 포함된 설명 가능한 검색을 제공하며, 여러 시스템에 걸쳐 워크플로를 자동화하는 오픈소스이자 셀프 호스팅이 가능한 AI 네이티브 실행 계층입니다.


## 주요 기능

- 📝 **설명 가능한 답변:** PipesHub는 원본 문서의 정확한 블록 단위 인용과 함께 근거 있는 답변을 제공합니다.
- 🔒 **권한 인식 검색:** 소스 수준의 접근 제어를 적용하여 사용자는 권한이 있는 내용만 볼 수 있습니다.
- 🕸️ **지식 그래프 검색:** 기업 데이터 전반의 관계를 포착하는 그래프 기반 검색.
- 🔌 **엔터프라이즈 커넥터:** 실시간 및 예약 색인을 기본 제공하는 30개 이상의 커넥터.
- 🔍 **통합 검색, 심층 리서치, 에이전트:** 검색, Q&A, 심층 리서치, 웹 검색, AI 에이전트를 하나의 컨텍스트 계층에서 제공합니다.
- 📊 **아티팩트 및 코드 실행:** 안전한 실행 샌드박스에서 보고서, 차트, 대시보드를 생성합니다.
- 🎙️ **멀티모달 지원:** 이미지, 다이어그램, 스캔 파일 이해는 물론 음성 기반 상호작용도 지원합니다.
- 🤖 **노코드 에이전트와 액션:** 에이전트를 시각적으로 구축하고 기업 도구 전반에서 액션을 실행합니다.
- 🧠 **자체 모델 사용(BYOM), 완전한 셀프 호스팅:** 어떤 LLM 제공업체든 사용 가능하며, VPC에 배포되어 데이터가 인프라를 떠나지 않습니다.
- 🛠️ **개발자 우선, 확장 가능:** API, SDK, MCP 도구, 커스텀 커넥터, 독립적으로 확장 가능한 서비스.

## 🚀 배포 가이드

PipesHub(업무용 AI 플랫폼)는 로컬에서 실행하거나 Docker Compose를 사용하여 클라우드에 배포할 수 있습니다.
**참고**: PipesHub를 클라우드 서버에 배포하는 경우 반드시 HTTPS 엔드포인트를 사용하세요. PipesHub는 더 엄격한 보안 검사를 수행하므로, 애플리케이션이 HTTP로 제공될 때 브라우저는 일부 요청을 차단합니다.
Cloudflare, Nginx, Traefik 같은 리버스 프록시를 사용하여 SSL/TLS를 종료하고 유효한 HTTPS 인증서를 제공할 수 있습니다.
HTTP로 접속할 때 배포된 PipesHub에서 흰 화면이 보인다면, 이것이 원인일 가능성이 높습니다. 더 엄격한 보안 검사로 인해 프런트엔드가 로드를 거부합니다.

---

### 📦 프로덕션 배포

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 개발자 배포 빌드

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

메인 `Dockerfile`은 `pipeshubai/pipeshub-ai-base:python-deps`와 `pipeshubai/pipeshub-ai-base:runtime`에서 사전 빌드된 레이어를 가져옵니다(빌드/푸시 명령은 저장소 루트의 [`Dockerfile.base`](../../../Dockerfile.base) 참조). 대신 로컬 태그를 사용하려면 환경 변수 또는 compose 빌드 인자에서 `PYTHON_DEPS_IMAGE`와 `RUNTIME_BASE_IMAGE`를 설정하세요.

## 자주 묻는 질문

### PipesHub란 무엇인가요?

PipesHub는 기업 지식을 연결하고, 인용이 포함된 설명 가능한 검색을 제공하며, 여러 시스템에 걸쳐 워크플로를 자동화하는 오픈소스이자 셀프 호스팅이 가능한 AI 네이티브 실행 계층입니다. 검색, Q&A, 심층 리서치, 웹 검색, AI 에이전트를 위한 통합 컨텍스트 계층을 제공합니다.

### PipesHub는 다른 업무용 AI 도구와 어떻게 다른가요?

PipesHub는 완전한 오픈소스(Apache 2.0)이며 셀프 호스팅이 가능하여 데이터가 인프라를 떠나지 않습니다. 소스 수준의 접근 제어를 적용하는 권한 인식 검색을 제공하며, 원본 문서의 정확한 블록 단위 인용과 함께 설명 가능한 답변을 제공합니다.

### PipesHub는 어떤 커넥터를 지원하나요?

PipesHub는 실시간 및 예약 색인을 지원하는 30개 이상의 엔터프라이즈 커넥터를 갖추고 있습니다. PDF, Docx, XLSX, PPTX, CSV, Markdown, HTML, Google Docs/Sheets/Slides, 이미지, 오디오, 비디오 같은 파일 형식을 지원합니다.

### PipesHub는 어떻게 배포하나요?

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

참고: 클라우드 배포에는 HTTPS를 사용하세요. HTTP는 프런트엔드 보안 차단을 유발할 수 있습니다.

### PipesHub는 어떤 LLM 제공업체를 지원하나요?

PipesHub는 "자체 모델 사용(Bring Your Own Model)" 방식으로, 어떤 LLM 제공업체든 사용할 수 있습니다. 원하는 모델을 VPC에 배포하세요. 기술 스택에는 LLM 파이프라인과 워크플로를 위한 LangChain과 LangGraph가 포함됩니다.

### 지식 그래프 검색 기능이란 무엇인가요?

PipesHub는 기업 데이터 전반의 관계를 포착하는 그래프 기반 검색을 사용합니다. 그래프 데이터베이스로 Neo4j 또는 ArangoDB를 사용하고, 벡터 유사도 검색을 위해 Qdrant와 결합합니다.

### PipesHub에 MCP 서버가 있나요?

네. PipesHub는 MCP 호환 클라이언트와의 통합을 위한 MCP 서버를 제공합니다. 저장소: [pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/).

### 어떤 SDK를 사용할 수 있나요?

PipesHub는 다음 SDK를 제공합니다:
- **Python**: [pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript**: [pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go**: [pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### 코딩 없이 AI 에이전트를 구축할 수 있나요?

네. PipesHub에는 노코드 에이전트 빌더가 있습니다. 코드를 작성하지 않고도 에이전트를 시각적으로 구축하고 기업 도구 전반에서 액션을 실행할 수 있습니다.

### 멀티모달 지원이란 무엇인가요?

PipesHub는 이미지, 다이어그램, 스캔 파일 이해는 물론 음성 기반 상호작용을 지원합니다. 문서 파싱에는 Docling과 PyMuPDF를 사용하고, 스캔된 PDF의 OCR에는 Azure Document Intelligence 또는 멀티모달 LLM(VLM)을 사용합니다.

### 배포 문제는 어떻게 해결하나요?

1. 클라우드 배포에 HTTPS가 구성되어 있는지 확인하세요
2. Docker compose 로그를 확인하세요: `docker compose logs`
3. env.template의 환경 변수를 확인하세요
4. 자세한 가이드는 [docs.pipeshub.com](https://docs.pipeshub.com/)을 참조하세요
