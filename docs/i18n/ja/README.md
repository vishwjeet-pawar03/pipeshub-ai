<h2 id="about-pipeshub">PipesHub — 説明可能で拡張可能</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> は、企業のナレッジを接続し、引用付きの説明可能な検索を提供し、各システムをまたいでワークフローを自動化する、オープンソースかつセルフホスト可能な AI ネイティブの実行レイヤーです。


## 機能

- 📝 **説明可能な回答：** PipesHub は、元のドキュメントへの正確なブロック単位の引用を伴う、根拠に基づいた回答を提供します。
- 🔒 **権限を考慮した検索：** ソースレベルのアクセス制御を適用し、ユーザーは権限のある範囲のみを閲覧できます。
- 🕸️ **ナレッジグラフ検索：** 企業データ間の関係性を捉える、グラフベースの検索。
- 🔌 **エンタープライズコネクタ：** リアルタイムおよびスケジュール索引に対応した 30 以上のコネクタを標準搭載。
- 🔍 **統合検索・ディープリサーチ・エージェント：** 検索、Q&A、ディープリサーチ、Web 検索、AI エージェントを単一のコンテキストレイヤー上で提供。
- 📊 **アーティファクトとコード実行：** 安全な実行サンドボックス内でレポート、グラフ、ダッシュボードを生成。
- 🎙️ **マルチモーダル対応：** 画像・図・スキャンファイルの理解に加え、音声ベースの操作にも対応。
- 🤖 **ノーコードのエージェントとアクション：** エージェントを視覚的に構築し、企業向けツール全体でアクションを実行。
- 🧠 **自前のモデルを使用可能、完全セルフホスト：** 任意の LLM プロバイダーを、あなたの VPC 内にデプロイ — データはインフラから外に出ません。
- 🛠️ **開発者ファーストで拡張可能：** API、SDK、MCP ツール、カスタムコネクタ、独立してスケール可能なサービス。

## 🚀 デプロイガイド

PipesHub（職場向け AI プラットフォーム）は、ローカルで実行することも、Docker Compose を使用してクラウドにデプロイすることもできます。
**注意**：PipesHub をクラウドサーバーにデプロイする場合は、必ず HTTPS エンドポイントを使用してください。PipesHub はより厳格なセキュリティチェックを実施するため、アプリケーションが HTTP で提供される場合、ブラウザは一部のリクエストをブロックします。
Cloudflare、Nginx、Traefik などのリバースプロキシを使用して SSL/TLS を終端し、有効な HTTPS 証明書を提供できます。
HTTP 経由でアクセスした際にデプロイ後の PipesHub が白い画面になる場合、これが原因である可能性が高いです。より厳格なセキュリティチェックにより、フロントエンドの読み込みが拒否されます。

### 📦 本番デプロイ

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 開発者向けデプロイビルド

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

メインの `Dockerfile` は、`pipeshubai/pipeshub-ai-base:python-deps` と `pipeshubai/pipeshub-ai-base:runtime` からビルド済みのレイヤーを取得します（ビルド／プッシュのコマンドはリポジトリルートの [`Dockerfile.base`](../../../Dockerfile.base) を参照）。代わりにローカルタグを使用するには、環境変数または compose のビルド引数で `PYTHON_DEPS_IMAGE` と `RUNTIME_BASE_IMAGE` を設定してください。

## FAQ

### PipesHub とは何ですか？

PipesHub は、企業のナレッジを接続し、引用付きの説明可能な検索を提供し、各システムをまたいでワークフローを自動化する、オープンソースかつセルフホスト可能な AI ネイティブの実行レイヤーです。検索、Q&A、ディープリサーチ、Web 検索、AI エージェントのための統一されたコンテキストレイヤーを提供します。

### PipesHub は他の職場向け AI ツールとどう違いますか？

PipesHub は完全にオープンソース（Apache 2.0）でセルフホスト可能であり、あなたのデータはインフラの外に出ません。ソースレベルのアクセス制御を適用する権限考慮型の検索を備え、元のドキュメントへの正確なブロック単位の引用を伴う説明可能な回答を提供します。

### PipesHub はどのようなコネクタに対応していますか？

PipesHub はリアルタイムおよびスケジュール索引に対応した 30 以上のエンタープライズコネクタを備えています。PDF、Docx、XLSX、PPTX、CSV、Markdown、HTML、Google Docs/Sheets/Slides、画像、音声、動画などのファイル形式に対応しています。

### PipesHub はどうやってデプロイしますか？

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

注意：クラウドへのデプロイには HTTPS を使用してください。HTTP ではフロントエンドのセキュリティブロックが発生する場合があります。

### PipesHub はどの LLM プロバイダーに対応していますか？

PipesHub は「自前のモデルを使用（Bring Your Own Model）」方式で、任意の LLM プロバイダーを使用できます。お好みのモデルを VPC 内にデプロイしてください。技術スタックには、LLM パイプラインとワークフローのための LangChain と LangGraph が含まれます。

### ナレッジグラフ検索機能とは何ですか？

PipesHub は、企業データ間の関係性を捉えるグラフベースの検索を使用します。グラフデータベースとして Neo4j または ArangoDB を使用し、ベクトル類似度検索のための Qdrant と組み合わせます。

### PipesHub に MCP サーバーはありますか？

あります。PipesHub は、MCP 対応の任意のクライアントと統合するための MCP サーバーを提供します。リポジトリ：[pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/)。

### どのような SDK が利用できますか？

PipesHub は以下の SDK を提供しています：
- **Python**：[pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript**：[pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go**：[pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### コードを書かずに AI エージェントを構築できますか？

はい。PipesHub にはノーコードのエージェントビルダーがあります。コードを書くことなく、エージェントを視覚的に構築し、企業向けツール全体でアクションを実行できます。

### マルチモーダル対応とは何ですか？

PipesHub は、画像・図・スキャンファイルの理解に加え、音声ベースの操作に対応しています。ドキュメント解析には Docling と PyMuPDF を使用し、スキャンされた PDF の OCR には Azure Document Intelligence またはマルチモーダル LLM（VLM）を使用します。

### デプロイの問題をどうトラブルシューティングしますか？

1. クラウドへのデプロイで HTTPS が構成されていることを確認する
2. Docker compose のログを確認する：`docker compose logs`
3. env.template の環境変数を確認する
4. 詳細なガイドは [docs.pipeshub.com](https://docs.pipeshub.com/) を参照する
