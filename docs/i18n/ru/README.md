<h2 id="about-pipeshub">PipesHub — объяснимый и расширяемый</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> — это ИИ-ориентированный слой выполнения с открытым исходным кодом и возможностью self-hosting, который связывает корпоративные знания, обеспечивает объяснимый поиск со ссылками на источники и автоматизирует рабочие процессы в ваших системах.


## Возможности

- 📝 **Объяснимые ответы:** PipesHub предоставляет обоснованные ответы с точными ссылками на конкретные блоки исходных документов.
- 🔒 **Поиск с учётом прав доступа:** Применяет контроль доступа на уровне источника, поэтому пользователи видят только то, к чему у них есть права.
- 🕸️ **Извлечение по графу знаний:** Извлечение на основе графа, фиксирующее связи между корпоративными данными.
- 🔌 **Корпоративные коннекторы:** Более 30 коннекторов с индексацией в реальном времени и по расписанию из коробки.
- 🔍 **Единый поиск, глубокие исследования и агенты:** Поиск, вопросы и ответы, глубокие исследования, веб-поиск и ИИ-агенты на одном контекстном слое.
- 📊 **Артефакты и выполнение кода:** Генерируйте отчёты, диаграммы и дашборды в безопасной песочнице выполнения.
- 🎙️ **Мультимодальная поддержка:** Понимание изображений, схем и отсканированных файлов, а также голосовое взаимодействие.
- 🤖 **No-code агенты и действия:** Создавайте агентов визуально и выполняйте действия в корпоративных инструментах.
- 🧠 **Используйте свою модель, полностью self-hosted:** Любой провайдер LLM, развёрнутый в вашем VPC — данные никогда не покидают вашу инфраструктуру.
- 🛠️ **Ориентация на разработчиков и расширяемость:** API, SDK, инструменты MCP, пользовательские коннекторы и независимо масштабируемые сервисы.

## 🚀 Руководство по развёртыванию

PipesHub (платформу корпоративного ИИ) можно запустить локально или развернуть в облаке с помощью Docker Compose.
**Примечание**: Если вы развёртываете PipesHub на облачном сервере, обязательно используйте HTTPS-эндпоинт. PipesHub применяет более строгие проверки безопасности, и браузеры будут блокировать некоторые запросы, когда приложение обслуживается по HTTP.
Вы можете использовать обратный прокси, например Cloudflare, Nginx или Traefik, чтобы терминировать SSL/TLS и предоставить действительный сертификат HTTPS.
Если после развёртывания PipesHub вы видите белый экран при доступе по HTTP, вероятнее всего, причина именно в этом. Фронтенд откажется загружаться из-за более строгих проверок безопасности.

---

### 📦 Развёртывание в продакшене

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 Сборка для развёртывания разработчиками

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

Основной `Dockerfile` загружает предварительно собранные слои из `pipeshubai/pipeshub-ai-base:python-deps` и `pipeshubai/pipeshub-ai-base:runtime` (команды сборки/публикации см. в [`Dockerfile.base`](../../../Dockerfile.base) в корне репозитория). Чтобы вместо этого использовать локальные теги, задайте `PYTHON_DEPS_IMAGE` и `RUNTIME_BASE_IMAGE` в окружении или в аргументах сборки compose.

## Часто задаваемые вопросы

### Что такое PipesHub?

PipesHub — это ИИ-ориентированный слой выполнения с открытым исходным кодом и возможностью self-hosting, который связывает корпоративные знания, обеспечивает объяснимый поиск со ссылками на источники и автоматизирует рабочие процессы в ваших системах. Он предоставляет единый контекстный слой для поиска, вопросов и ответов, глубоких исследований, веб-поиска и ИИ-агентов.

### Чем PipesHub отличается от других инструментов корпоративного ИИ?

PipesHub полностью открыт (Apache 2.0) и поддерживает self-hosting — ваши данные никогда не покидают вашу инфраструктуру. В нём есть поиск с учётом прав доступа, применяющий контроль доступа на уровне источника, и он предоставляет объяснимые ответы с точными ссылками на конкретные блоки исходных документов.

### Какие коннекторы поддерживает PipesHub?

PipesHub имеет более 30 корпоративных коннекторов с индексацией в реальном времени и по расписанию. Он поддерживает такие форматы файлов, как PDF, Docx, XLSX, PPTX, CSV, Markdown, HTML, Google Docs/Sheets/Slides, изображения, аудио и видео.

### Как развернуть PipesHub?

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

Примечание: Используйте HTTPS для облачных развёртываний. HTTP может привести к блокировкам безопасности на фронтенде.

### Каких провайдеров LLM поддерживает PipesHub?

PipesHub работает по принципу «Используйте свою модель» — вы можете использовать любого провайдера LLM. Разверните в своём VPC предпочитаемые модели. Технологический стек включает LangChain и LangGraph для конвейеров и рабочих процессов LLM.

### Что такое функция извлечения по графу знаний?

PipesHub использует извлечение на основе графа, которое фиксирует связи между корпоративными данными. В качестве графовых баз данных используются Neo4j или ArangoDB в сочетании с Qdrant для поиска по векторному сходству.

### Есть ли у PipesHub MCP-сервер?

Да. PipesHub предоставляет MCP-сервер для интеграции с любым MCP-совместимым клиентом. Репозиторий: [pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/).

### Какие SDK доступны?

PipesHub предоставляет SDK для:
- **Python**: [pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript**: [pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go**: [pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### Можно ли создавать ИИ-агентов без программирования?

Да. В PipesHub есть no-code конструктор агентов. Вы можете создавать агентов визуально и выполнять действия в корпоративных инструментах, не написав ни строки кода.

### Что такое мультимодальная поддержка?

PipesHub поддерживает понимание изображений, схем и отсканированных файлов, а также голосовое взаимодействие. Для разбора документов используются Docling и PyMuPDF, а для OCR отсканированных PDF — Azure Document Intelligence или мультимодальная LLM (VLM).

### Как устранять проблемы с развёртыванием?

1. Убедитесь, что для облачных развёртываний настроен HTTPS
2. Проверьте логи Docker compose: `docker compose logs`
3. Проверьте переменные окружения в env.template
4. Обратитесь к [docs.pipeshub.com](https://docs.pipeshub.com/) за подробными руководствами
