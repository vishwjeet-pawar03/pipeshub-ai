<h2 id="about-pipeshub">PipesHub — Explicable et extensible</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> est une couche d'exécution native pour l'IA, open source et auto-hébergeable, qui connecte les connaissances de l'entreprise, fournit une recherche explicable avec citations et automatise les flux de travail à travers vos systèmes.


## Fonctionnalités

- 📝 **Réponses explicables :** PipesHub fournit des réponses étayées avec des citations précises par bloc vers les documents d'origine.
- 🔒 **Recherche tenant compte des permissions :** Applique des contrôles d'accès au niveau de la source afin que les utilisateurs ne voient que ce à quoi ils sont autorisés.
- 🕸️ **Récupération par graphe de connaissances :** Récupération fondée sur les graphes, qui capture les relations entre les données de l'entreprise.
- 🔌 **Connecteurs d'entreprise :** Plus de 30 connecteurs avec indexation en temps réel et planifiée, prêts à l'emploi.
- 🔍 **Recherche unifiée, recherche approfondie et agents :** Recherche, questions-réponses, recherche approfondie, recherche web et agents IA sur une seule couche de contexte.
- 📊 **Artefacts et exécution de code :** Générez des rapports, des graphiques et des tableaux de bord dans un bac à sable d'exécution sécurisé.
- 🎙️ **Prise en charge multimodale :** Compréhension des images, des schémas et des fichiers numérisés, ainsi qu'une interaction vocale.
- 🤖 **Agents et actions no-code :** Créez des agents visuellement et exécutez des actions à travers les outils de l'entreprise.
- 🧠 **Apportez votre propre modèle, entièrement auto-hébergeable :** Tout fournisseur de LLM, déployé dans votre VPC — les données ne quittent jamais votre infrastructure.
- 🛠️ **Pensé pour les développeurs et extensible :** API, SDK, outils MCP, connecteurs personnalisés et services pouvant évoluer indépendamment.

## 🚀 Guide de déploiement

PipesHub (la plateforme d'IA pour le travail) peut être exécuté localement ou déployé dans le cloud à l'aide de Docker Compose.
**Remarque** : Si vous déployez PipesHub sur un serveur cloud, assurez-vous d'utiliser un point de terminaison HTTPS. PipesHub applique des contrôles de sécurité plus stricts, et les navigateurs bloqueront certaines requêtes lorsque l'application est servie via HTTP.
Vous pouvez utiliser un proxy inverse comme Cloudflare, Nginx ou Traefik pour terminer le SSL/TLS et fournir un certificat HTTPS valide.
Si vous voyez un écran blanc après avoir déployé PipesHub en y accédant via HTTP, c'est probablement la cause. Le frontend refusera de se charger en raison des contrôles de sécurité plus stricts.

---

### 📦 Déploiement en production

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 Build de déploiement pour développeurs

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

Le `Dockerfile` principal récupère des couches pré-construites depuis `pipeshubai/pipeshub-ai-base:python-deps` et `pipeshubai/pipeshub-ai-base:runtime` (voir [`Dockerfile.base`](../../../Dockerfile.base) à la racine du dépôt pour les commandes de build/push). Pour utiliser des tags locaux à la place, définissez `PYTHON_DEPS_IMAGE` et `RUNTIME_BASE_IMAGE` dans l'environnement ou dans les arguments de build de compose.

## FAQ

### Qu'est-ce que PipesHub ?

PipesHub est une couche d'exécution native pour l'IA, open source et auto-hébergeable, qui connecte les connaissances de l'entreprise, fournit une recherche explicable avec citations et automatise les flux de travail à travers vos systèmes. Elle offre une couche de contexte unifiée pour la recherche, les questions-réponses, la recherche approfondie, la recherche web et les agents IA.

### En quoi PipesHub diffère-t-il des autres outils d'IA pour le travail ?

PipesHub est entièrement open source (Apache 2.0) et auto-hébergeable — vos données ne quittent jamais votre infrastructure. Il propose une recherche tenant compte des permissions qui applique des contrôles d'accès au niveau de la source, et fournit des réponses explicables avec des citations précises par bloc vers les documents d'origine.

### Quels connecteurs PipesHub prend-il en charge ?

PipesHub dispose de plus de 30 connecteurs d'entreprise avec indexation en temps réel et planifiée. Il prend en charge des formats de fichiers tels que PDF, Docx, XLSX, PPTX, CSV, Markdown, HTML, Google Docs/Sheets/Slides, images, audio et vidéo.

### Comment déployer PipesHub ?

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

Remarque : Utilisez HTTPS pour les déploiements dans le cloud. Le HTTP peut entraîner des blocages de sécurité au niveau du frontend.

### Quels fournisseurs de LLM PipesHub prend-il en charge ?

PipesHub fonctionne selon le principe « Apportez votre propre modèle » — vous pouvez utiliser n'importe quel fournisseur de LLM. Déployez dans votre VPC avec les modèles de votre choix. La stack technique inclut LangChain et LangGraph pour les pipelines et les workflows de LLM.

### Qu'est-ce que la fonctionnalité de récupération par graphe de connaissances ?

PipesHub utilise une récupération fondée sur les graphes qui capture les relations entre les données de l'entreprise. Il utilise Neo4j ou ArangoDB comme bases de données orientées graphe, combinées à Qdrant pour la recherche par similarité vectorielle.

### PipesHub dispose-t-il d'un serveur MCP ?

Oui. PipesHub fournit un serveur MCP pour l'intégration avec n'importe quel client compatible MCP. Dépôt : [pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/).

### Quels SDK sont disponibles ?

PipesHub fournit des SDK pour :
- **Python** : [pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript** : [pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go** : [pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### Puis-je créer des agents IA sans coder ?

Oui. PipesHub dispose d'un constructeur d'agents no-code. Vous pouvez créer des agents visuellement et exécuter des actions à travers les outils de l'entreprise sans écrire de code.

### Qu'est-ce que la prise en charge multimodale ?

PipesHub prend en charge la compréhension des images, des schémas et des fichiers numérisés, ainsi que l'interaction vocale. Il utilise Docling et PyMuPDF pour l'analyse des documents, et Azure Document Intelligence ou un LLM multimodal (VLM) pour l'OCR des PDF numérisés.

### Comment résoudre les problèmes de déploiement ?

1. Assurez-vous que HTTPS est configuré pour les déploiements dans le cloud
2. Vérifiez les journaux de Docker compose : `docker compose logs`
3. Vérifiez les variables d'environnement dans env.template
4. Consultez [docs.pipeshub.com](https://docs.pipeshub.com/) pour des guides détaillés
