<h2 id="about-pipeshub">PipesHub — Explicável e Extensível</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> é uma camada de execução nativa de IA, de código aberto e auto-hospedável, que conecta o conhecimento corporativo, oferece busca explicável com citações e automatiza fluxos de trabalho em todos os seus sistemas.


## Funcionalidades

- 📝 **Respostas explicáveis:** O PipesHub entrega respostas fundamentadas com citações precisas de blocos aos documentos originais.
- 🔒 **Busca com reconhecimento de permissões:** Aplica controles de acesso no nível da fonte para que os usuários vejam apenas o que têm autorização.
- 🕸️ **Recuperação por grafo de conhecimento:** Recuperação baseada em grafos que captura as relações entre os dados corporativos.
- 🔌 **Conectores corporativos:** Mais de 30 conectores com indexação em tempo real e agendada, prontos para uso.
- 🔍 **Busca unificada, pesquisa aprofundada e agentes:** Busca, perguntas e respostas, pesquisa aprofundada, busca na web e agentes de IA sobre uma única camada de contexto.
- 📊 **Artefatos e execução de código:** Gere relatórios, gráficos e painéis em um sandbox de execução seguro.
- 🎙️ **Suporte multimodal:** Compreensão de imagens, diagramas e arquivos digitalizados, além de interação por voz.
- 🤖 **Agentes e ações sem código:** Crie agentes visualmente e execute ações nas ferramentas corporativas.
- 🧠 **Use seu próprio modelo, totalmente auto-hospedável:** Qualquer provedor de LLM, implantado na sua VPC — os dados nunca saem da sua infraestrutura.
- 🛠️ **Voltado a desenvolvedores e extensível:** APIs, SDKs, ferramentas MCP, conectores personalizados e serviços escaláveis de forma independente.

## 🚀 Guia de implantação

O PipesHub (a plataforma de IA para o trabalho) pode ser executado localmente ou implantado na nuvem usando o Docker Compose.
**Observação**: Se você for implantar o PipesHub em um servidor na nuvem, certifique-se de usar um endpoint HTTPS. O PipesHub aplica verificações de segurança mais rígidas, e os navegadores bloquearão certas requisições quando a aplicação for servida por HTTP.
Você pode usar um proxy reverso como Cloudflare, Nginx ou Traefik para terminar o SSL/TLS e fornecer um certificado HTTPS válido.
Se você vir uma tela branca após implantar o PipesHub ao acessá-lo por HTTP, esta é provavelmente a causa. O frontend se recusará a carregar devido às verificações de segurança mais rígidas.

---

### 📦 Implantação em produção

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 Build de implantação para desenvolvedores

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

O `Dockerfile` principal obtém camadas pré-construídas de `pipeshubai/pipeshub-ai-base:python-deps` e `pipeshubai/pipeshub-ai-base:runtime` (consulte [`Dockerfile.base`](../../../Dockerfile.base) na raiz do repositório para os comandos de build/push). Para usar tags locais em vez disso, defina `PYTHON_DEPS_IMAGE` e `RUNTIME_BASE_IMAGE` no ambiente ou nos argumentos de build do compose.

## Perguntas frequentes

### O que é o PipesHub?

O PipesHub é uma camada de execução nativa de IA, de código aberto e auto-hospedável, que conecta o conhecimento corporativo, oferece busca explicável com citações e automatiza fluxos de trabalho em todos os seus sistemas. Ele fornece uma camada de contexto unificada para busca, perguntas e respostas, pesquisa aprofundada, busca na web e agentes de IA.

### Como o PipesHub se diferencia de outras ferramentas de IA para o trabalho?

O PipesHub é totalmente de código aberto (Apache 2.0) e auto-hospedável — seus dados nunca saem da sua infraestrutura. Ele conta com busca com reconhecimento de permissões que aplica controles de acesso no nível da fonte e entrega respostas explicáveis com citações precisas de blocos aos documentos originais.

### Quais conectores o PipesHub suporta?

O PipesHub possui mais de 30 conectores corporativos com indexação em tempo real e agendada. Ele suporta formatos de arquivo como PDF, Docx, XLSX, PPTX, CSV, Markdown, HTML, Google Docs/Sheets/Slides, imagens, áudio e vídeo.

### Como eu implanto o PipesHub?

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

Observação: Use HTTPS para implantações na nuvem. O HTTP pode causar bloqueios de segurança no frontend.

### Quais provedores de LLM o PipesHub suporta?

O PipesHub é "Use seu próprio modelo" — você pode usar qualquer provedor de LLM. Implante na sua VPC com os modelos de sua preferência. O stack tecnológico inclui LangChain e LangGraph para pipelines e fluxos de trabalho de LLM.

### O que é o recurso de recuperação por grafo de conhecimento?

O PipesHub usa recuperação baseada em grafos que captura as relações entre os dados corporativos. Ele usa Neo4j ou ArangoDB como bancos de dados de grafos, combinados com o Qdrant para busca por similaridade vetorial.

### O PipesHub tem um servidor MCP?

Sim. O PipesHub oferece um servidor MCP para integração com qualquer cliente compatível com MCP. Repositório: [pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/).

### Quais SDKs estão disponíveis?

O PipesHub oferece SDKs para:
- **Python**: [pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript**: [pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go**: [pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### Posso criar agentes de IA sem programar?

Sim. O PipesHub tem um construtor de agentes sem código. Você pode criar agentes visualmente e executar ações nas ferramentas corporativas sem escrever código.

### O que é o suporte multimodal?

O PipesHub suporta a compreensão de imagens, diagramas e arquivos digitalizados, além da interação por voz. Ele usa Docling e PyMuPDF para análise de documentos, e Azure Document Intelligence ou um LLM multimodal (VLM) para o OCR de PDFs digitalizados.

### Como soluciono problemas de implantação?

1. Certifique-se de que o HTTPS está configurado para implantações na nuvem
2. Verifique os logs do Docker compose: `docker compose logs`
3. Confira as variáveis de ambiente em env.template
4. Consulte [docs.pipeshub.com](https://docs.pipeshub.com/) para guias detalhados
