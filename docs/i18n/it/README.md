<h2 id="about-pipeshub">PipesHub — Spiegabile ed Estensibile</h2>

<strong>[PipesHub](https://www.pipeshub.com/)</strong> è un livello di esecuzione nativo per l'IA, open source e auto-ospitabile, che collega la conoscenza aziendale, offre una ricerca spiegabile con citazioni e automatizza i flussi di lavoro nei tuoi sistemi.


## Funzionalità

- 📝 **Risposte spiegabili:** PipesHub fornisce risposte fondate con citazioni precise per blocchi ai documenti originali.
- 🔒 **Ricerca consapevole dei permessi:** Applica controlli di accesso a livello di fonte, così gli utenti vedono solo ciò a cui sono autorizzati.
- 🕸️ **Recupero tramite grafo della conoscenza:** Recupero basato su grafi che cattura le relazioni tra i dati aziendali.
- 🔌 **Connettori aziendali:** Oltre 30 connettori con indicizzazione in tempo reale e pianificata, pronti all'uso.
- 🔍 **Ricerca unificata, ricerca approfondita e agenti:** Ricerca, domande e risposte, ricerca approfondita, ricerca sul web e agenti IA su un unico livello di contesto.
- 📊 **Artefatti ed esecuzione di codice:** Genera report, grafici e dashboard in una sandbox di esecuzione sicura.
- 🎙️ **Supporto multimodale:** Comprensione di immagini, diagrammi e file scansionati, oltre all'interazione vocale.
- 🤖 **Agenti e azioni no-code:** Crea agenti in modo visivo ed esegui azioni sugli strumenti aziendali.
- 🧠 **Porta il tuo modello, completamente auto-ospitabile:** Qualsiasi provider di LLM, distribuito nella tua VPC — i dati non lasciano mai la tua infrastruttura.
- 🛠️ **Pensato per gli sviluppatori ed estensibile:** API, SDK, strumenti MCP, connettori personalizzati e servizi scalabili in modo indipendente.

## 🚀 Guida al deployment

PipesHub (la piattaforma di IA per il lavoro) può essere eseguito localmente o distribuito sul cloud usando Docker Compose.
**Nota**: Se distribuisci PipesHub su un server cloud, assicurati di usare un endpoint HTTPS. PipesHub applica controlli di sicurezza più rigorosi e i browser bloccheranno alcune richieste quando l'applicazione è servita via HTTP.
Puoi usare un reverse proxy come Cloudflare, Nginx o Traefik per terminare SSL/TLS e fornire un certificato HTTPS valido.
Se vedi una schermata bianca dopo aver distribuito PipesHub accedendovi via HTTP, questa è probabilmente la causa. Il frontend rifiuterà di caricarsi a causa dei controlli di sicurezza più rigorosi.

---

### 📦 Deployment in produzione

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d

# 🛑 To stop the services
docker compose -f docker-compose.prod.yml -p pipeshub-ai down
```

### 📦 Build di deployment per sviluppatori

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git

# 📁 Navigate to the deployment folder
cd pipeshub-ai/deployment/docker-compose

# Set Optional Environment Variables
> 👉 Set Environment Variables for secrets, passwords, and the public URLs of the **Frontend** and **Connector** services
> _(Required for webhook notifications and real-time updates)_
> Refer to env.template

# 🚀 Start the local build deployment
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai up --build -d

# 🛑 To stop the services
docker compose -f docker-compose.build.neo4j.yml -p pipeshub-ai down
```

Il `Dockerfile` principale recupera layer precompilati da `pipeshubai/pipeshub-ai-base:python-deps` e `pipeshubai/pipeshub-ai-base:runtime` (consulta [`Dockerfile.base`](../../../Dockerfile.base) nella radice del repository per i comandi di build/push). Per usare invece tag locali, imposta `PYTHON_DEPS_IMAGE` e `RUNTIME_BASE_IMAGE` nell'ambiente o negli argomenti di build di compose.

## Domande frequenti

### Che cos'è PipesHub?

PipesHub è un livello di esecuzione nativo per l'IA, open source e auto-ospitabile, che collega la conoscenza aziendale, offre una ricerca spiegabile con citazioni e automatizza i flussi di lavoro nei tuoi sistemi. Fornisce un livello di contesto unificato per ricerca, domande e risposte, ricerca approfondita, ricerca sul web e agenti IA.

### In cosa si differenzia PipesHub dagli altri strumenti di IA per il lavoro?

PipesHub è completamente open source (Apache 2.0) e auto-ospitabile — i tuoi dati non lasciano mai la tua infrastruttura. Offre una ricerca consapevole dei permessi che applica controlli di accesso a livello di fonte e fornisce risposte spiegabili con citazioni precise per blocchi ai documenti originali.

### Quali connettori supporta PipesHub?

PipesHub dispone di oltre 30 connettori aziendali con indicizzazione in tempo reale e pianificata. Supporta formati di file come PDF, Docx, XLSX, PPTX, CSV, Markdown, HTML, Google Docs/Sheets/Slides, immagini, audio e video.

### Come si distribuisce PipesHub?

```bash
# Clone the repository
git clone https://github.com/pipeshub-ai/pipeshub-ai.git
cd pipeshub-ai/deployment/docker-compose

# Set Environment Variables (refer to env.template)
# Start production deployment
docker compose -f docker-compose.prod.yml -p pipeshub-ai up -d
```

Nota: Usa HTTPS per i deployment sul cloud. L'HTTP può causare blocchi di sicurezza nel frontend.

### Quali provider di LLM supporta PipesHub?

PipesHub adotta il principio "Porta il tuo modello" — puoi usare qualsiasi provider di LLM. Distribuisci nella tua VPC con i modelli che preferisci. Lo stack tecnologico include LangChain e LangGraph per le pipeline e i flussi di lavoro di LLM.

### Che cos'è la funzionalità di recupero tramite grafo della conoscenza?

PipesHub usa un recupero basato su grafi che cattura le relazioni tra i dati aziendali. Utilizza Neo4j o ArangoDB come database a grafo, combinati con Qdrant per la ricerca per similarità vettoriale.

### PipesHub ha un server MCP?

Sì. PipesHub fornisce un server MCP per l'integrazione con qualsiasi client compatibile con MCP. Repository: [pipeshub-ai/mcp-server](https://github.com/pipeshub-ai/mcp-server/).

### Quali SDK sono disponibili?

PipesHub fornisce SDK per:
- **Python**: [pipeshub-ai/pipeshub-sdk-python](https://github.com/pipeshub-ai/pipeshub-sdk-python)
- **TypeScript**: [pipeshub-ai/pipeshub-sdk-typescript](https://github.com/pipeshub-ai/pipeshub-sdk-typescript)
- **Go**: [pipeshub-ai/pipeshub-sdk-go](https://github.com/pipeshub-ai/pipeshub-sdk-go)

### Posso creare agenti IA senza programmare?

Sì. PipesHub ha un costruttore di agenti no-code. Puoi creare agenti in modo visivo ed eseguire azioni sugli strumenti aziendali senza scrivere codice.

### Che cos'è il supporto multimodale?

PipesHub supporta la comprensione di immagini, diagrammi e file scansionati, oltre all'interazione vocale. Usa Docling e PyMuPDF per l'analisi dei documenti e Azure Document Intelligence o un LLM multimodale (VLM) per l'OCR dei PDF scansionati.

### Come risolvo i problemi di deployment?

1. Assicurati che HTTPS sia configurato per i deployment sul cloud
2. Controlla i log di Docker compose: `docker compose logs`
3. Verifica le variabili d'ambiente in env.template
4. Consulta [docs.pipeshub.com](https://docs.pipeshub.com/) per guide dettagliate
